Project

Capstone is a web application backed by REpresentational State Transfer API. REST is an architectural style. It uses the HTTP methods (GET, POST, PUT, PATCH and DELETE) to conduct business. It always uses HTTP protocol for exchange of data. The format for data can be XML, TEXT or JSON. JSON (JavaScript Object Notation) is the preferred format.

Web services provide a standardized way of providing interoperability between disparate applications. This means the client and the server are completely decoupled from each other. They communicate using an Internet protocol.

Capstone is implemented using Java, JDBC and Spring Boot, HTML5, JQuery and (Bootstrap CSS). Spring offers transaction management and security and a very easy way of bean injection (as opposed to lookup in JEE). We chose to implement our application as a REST based web service. Different client applications ( implemented in different programming languages) could connect to it.

This application will serve to help Humber Radio Department to manage its coop students. The application offers the following use cases:

  • Displaying static shift information
  • Displaying employee information
  • Adding employee
  • Clocking an employee in
  • Clocking an employee out
  • Finding shifts for employees (given range of dates)
